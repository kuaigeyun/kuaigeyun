# RiverEdge SaaS 多租户框架 - 命名规范与哲学

## 🌿 命名哲学

**RiverEdge SaaS 多租户框架 (RiverEdge SaaS Multi-tenant Framework)** 采用**自然/植物生态**的命名哲学，将系统架构比作一个完整的生态系统：

```
RiverEdge 生态系统
├── core (根)      - 核心系统，提供基础支撑
├── shell (壳)     - 框架外壳，承载应用
├── seeds (种子)   - 应用插件，可生长的功能模块
├── land (土地)    - 着陆页，展示和生长的土壤
└── leaf (叶子)   - 移动端，轻量灵活的终端
```

## 📦 模块命名说明

### riveredge-core（核心系统）

**命名含义**：
- **Core** = 核心/根
- 如同植物的根系，提供基础支撑和养分
- 是整个系统的根基和基础

**职责**：
- 后端核心系统
- 提供 API 服务
- 数据持久化
- 认证授权

### riveredge-shell（前端框架）

**命名含义**：
- **Shell** = 外壳/框架
- 如同植物的外壳，保护和支持内部结构
- 承载和运行应用的基础框架

**职责**：
- 前端基础框架
- 插件加载器
- 路由管理
- 状态管理

### riveredge-seeds（应用插件）

**命名含义**：
- **Seeds** = 种子
- 如同植物的种子，可以生长成不同的功能模块
- 每个 seed 都是独立的、可生长的应用

**职责**：
- 独立的业务应用模块
- 以插件形式加载
- 包含前后端代码
- 可独立开发和部署

### riveredge-land（着陆页/官网）

**命名含义**：
- **Land** = 土地/着陆
- 如同植物的生长土壤，是展示和生长的平台
- 是用户"着陆"的第一站，也是系统对外展示的窗口
- 提供"土壤"让系统功能得以展示和生长

**职责**：
- SaaS 软件介绍主页
- 客户简易官网首页
- 产品展示和营销页面
- 系统对外展示窗口

**内涵匹配度**：✅ **高度匹配**
- 土地是种子生长的基础，land 是系统展示的基础
- 用户"着陆"在 land 上，了解系统功能
- 与 seeds（种子）形成完美的生态关系

### riveredge-leaf（移动端应用）

**命名含义**：
- **Leaf** = 叶子
- 如同植物的叶子，轻量、灵活、可移动
- 叶子是植物与外界交互的界面，移动端是系统与用户移动交互的界面
- 叶子可以随风而动，移动端可以随时随地被访问

**职责**：
- 移动端应用
- 生产现场数据录入
- 移动端工单查看
- 设备巡检应用

**内涵匹配度**：✅ **高度匹配**
- 叶子轻量灵活，移动端也是轻量灵活的
- 叶子是植物的一部分但可以独立存在，移动端是系统的一部分但可以独立部署
- 叶子与外界交互，移动端与用户移动交互
- 与 core（根）、shell（壳）、seeds（种子）形成完整的植物生态

## ✅ 命名建议

### 推荐命名

**完全采用您提议的命名**：

```
riveredge/
├── riveredge-core/      # 核心系统（根）
├── riveredge-shell/     # 前端框架（壳）
├── riveredge-seeds/     # 应用插件（种子）
├── riveredge-land/      # 着陆页/官网（土地）
└── riveredge-leaf/      # 移动端应用（叶子）
```

### 命名优势

1. **哲学一致性**：所有命名都遵循自然/植物生态的哲学
2. **内涵丰富**：每个命名都有深刻的生态含义
3. **易于理解**：命名直观，易于记忆和理解
4. **扩展性强**：未来可以继续使用生态相关的命名（如 branch、root、flower 等）

### 命名关系图

```
RiverEdge 生态系统

land (土地)
  ↓ 展示和生长
seeds (种子) → 在 land 上展示
  ↓ 加载到
shell (壳) → 承载 seeds
  ↓ 依赖
core (根) → 提供基础支撑
  ↓ 延伸
leaf (叶子) → 轻量灵活的移动端
```

## 📝 命名规范总结

### 已确定的命名

- ✅ **riveredge-core**: 后端核心系统（根）
- ✅ **riveredge-shell**: 前端基础框架（壳）
- ✅ **riveredge-seeds**: 应用插件集合（种子）
- ✅ **riveredge-land**: 着陆页/官网（土地）
- ✅ **riveredge-leaf**: 移动端应用（叶子）

### 命名特点

1. **统一前缀**: 所有模块使用 `riveredge-` 前缀
2. **生态主题**: 所有命名都围绕自然/植物生态
3. **简洁明了**: 单词简短，易于记忆
4. **内涵丰富**: 每个命名都有深刻的生态含义

## 🎨 命名哲学延伸

如果未来需要扩展，可以考虑以下命名：

- **riveredge-branch**: 分支系统（如多租户分支）
- **riveredge-root**: 根服务（如基础服务）
- **riveredge-flower**: 展示层（如报表展示）
- **riveredge-fruit**: 成果系统（如数据分析）

但当前的核心命名已经非常完整和优雅！
